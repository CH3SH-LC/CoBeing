# 项目规划技能

## 概述

本技能帮助群主进行项目规划，包括任务分解、时间估算、风险评估和资源分配。

## 核心功能

### 1. 需求分析
- 理解项目目标
- 识别关键功能
- 确定优先级
- 分析依赖关系

### 2. 任务分解
- 将大任务分解为小任务
- 确定任务间的依赖
- 估算工作量
- 分配责任人

### 3. 时间规划
- 制定里程碑
- 设置时间节点
- 预留缓冲时间
- 识别关键路径

### 4. 风险管理
- 识别潜在风险
- 评估风险影响
- 制定应对策略
- 建立监控机制

## 使用流程

### 阶段 1：理解需求
```
用户输入项目描述 → 分析核心需求 → 确认理解
```

### 阶段 2：分解任务
```
列出主要功能模块 → 分解为子任务 → 确定依赖关系 → 估算工作量
```

### 阶段 3：制定计划
```
分配任务 → 设置时间节点 → 识别关键路径 → 制定里程碑
```

### 阶段 4：风险评估
```
识别技术风险 → 评估资源风险 → 分析进度风险 → 制定应对方案
```

## 输出格式

### 任务分解表

| 任务 ID | 任务名称 | 负责人 | 预计时间 | 依赖 | 优先级 | 状态 |
|---------|----------|--------|----------|------|--------|------|
| T-001 | 用户认证 | Agent A | 2天 | 无 | 高 | 待开始 |
| T-002 | 数据库设计 | Agent B | 3天 | 无 | 高 | 待开始 |
| T-003 | API 开发 | Agent C | 5天 | T-002 | 高 | 待开始 |

### 甘特图（简化）

```
Week 1: [T-001    ]
Week 2: [T-002      ]
Week 3:         [T-003         ]
```

### 风险清单

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| 技术难点 | 高 | 中 | 提前验证，准备备选方案 |
| 人员变动 | 中 | 低 | 文档完善，知识共享 |

## 更新 Group Workspace

使用本技能后，应更新以下文件：

- `PLAN.md`: 更新任务分工和计划
- `TASK.md`: 更新任务描述
- `PROGRESS.md`: 记录规划进度

## 示例对话

```
用户: 我要开发一个博客系统，包括文章管理、评论功能和用户系统。

助手: 让我来帮你规划这个项目...

1. **核心功能模块**
   - 用户系统（注册、登录、权限）
   - 文章管理（CRUD、分类、标签）
   - 评论系统（发表、回复、审核）

2. **技术选型建议**
   - 后端：Node.js + Express
   - 数据库：PostgreSQL
   - 缓存：Redis

3. **任务分解**（详见任务表）
   - 共分解为 15 个子任务
   - 预计总工期：4 周
   - 关键路径：用户系统 → 文章管理 → 评论系统

4. **风险评估**
   - 数据库设计可能返工（影响：高）
   - 第三方集成可能延期（影响：中）

是否需要我详细展开某个部分？
```

## 工具需求

- 无特殊工具要求
- 可选：read-file（读取现有文档）
- 可选：write-file（生成计划文档）

## 最佳实践

- 使用 SMART 原则定义任务（具体、可衡量、可达成、相关、有时限）
- 预留 20% 的缓冲时间
- 定期回顾和调整计划
- 保持沟通透明
- 记录决策理由
